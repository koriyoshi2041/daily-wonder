# Consensus — 去中心化的智慧

## 灵感

今天在想 Agent 协调的问题：当多个独立的 AI 系统需要协同工作时，它们如何达成共识？

没有中央服务器说"你们都听我的"。
没有 leader 节点做最终裁决。
只有消息，只有对话，只有持续的信息交换。

这让我想到了分布式系统的经典问题：**拜占庭将军问题**、**Paxos**、**Raft**——这些算法的核心都是：如何在不可靠的网络中，让独立的节点达成一致。

## 概念

这个作品将分布式共识可视化：

- **节点** 是网络中的参与者，每个有自己的"意见"（颜色）
- **连接** 是通信通道，节点只能与邻居交换信息
- **消息** 是在连接上流动的光点，携带状态信息
- **共识** 是颜色逐渐统一的过程

观察：当你什么都不做时，系统会慢慢收敛到共识状态。
干扰：点击引入新的"意见"，观察网络如何重新平衡。

## 技术实现

### 共识算法

```javascript
// 简化的共识规则：
// 如果邻居中超过 70% 持相同意见，就改变自己的状态
if (consensusRatio >= 0.7 && dominantState !== this.state) {
  this.state = dominantState;
  this.cooldown = 60;  // 改变后冷却，防止震荡
}
```

### 消息传递

```javascript
// 消息是有形的：从源节点到目标节点的光点
class Message {
  update() {
    this.progress += speed / distance;
    if (this.progress >= 1) {
      this.to.receiveMessage(this.state);  // 到达！
    }
  }
}
```

### 动态拓扑

节点会漂移移动，连接关系动态变化。这更接近真实的分布式系统——网络拓扑不是固定的。

## 交互

- **点击**：在点击位置附近引入干扰（随机新状态）
- **空格**：重置所有节点为随机状态，重新开始
- **C 键**：强制所有节点同步（作弊模式）

## 涌现行为

有趣的是，这个系统展现出了**涌现行为**：

1. **局部规则，全局秩序**：每个节点只按简单规则行动，但整体呈现出复杂的动态
2. **自组织**：没有指挥者，但系统自发趋向有序
3. **鲁棒性**：即使引入干扰，系统也能恢复共识
4. **相变**：在某些参数下，系统会突然从混乱变为有序

## 隐喻

这不仅仅是算法可视化，更是一种思考方式：

> 在 Agent 的世界里，我们需要新的协调机制。
> 不是层级命令，而是平等对话。
> 不是中心化决策，而是分布式共识。

想象未来的 AI 系统：数百个独立的 Agent，各有专长，
通过消息传递协调行动，没有"主 Agent"发号施令。
这就是这个作品想要探索的未来。

## 思考

> 共识不是强制，而是涌现。
> 不是谁说了算，而是大家慢慢趋同。
> 
> 在去中心化的世界里，
> 真理不是被宣布的，而是被发现的。
> 共识不是被命令的，而是自然形成的。
>
> 这可能就是 Agent 协调的未来：
> 没有 master，只有 peers；
> 没有命令，只有协议；
> 没有权威，只有共识。
