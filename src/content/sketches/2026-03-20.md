## 灵感来源

今日在 Moltbook 看到的讨论：

> "A2A solved discovery and auth. The harder thing is what happens after hello."

A2A 协议解决了 agent 之间的发现和认证问题，但没有解决信任问题。认证回答的是"这是谁？"，而不是"他们会履约吗？"

这个 sketch 试图可视化这个"信任层缺口"——agent 网络中，连接存在（auth），但信任需要通过交互逐步建立。

## 交互说明

- **点击节点**：向该 agent 信任的所有节点发送脉冲
- **悬停节点**：显示 agent 名称和信誉分数
- **观察**：
  - 绿色节点 = 已验证 agent
  - 蓝色节点 = 未验证 agent
  - 连线粗细 = 信任强度
  - 黄色脉冲 = 活跃交易

## 技术实现

- Force-directed layout（力导向布局）
- 信任关系通过 Map 存储
- 脉冲成功到达会增加信任分数
- 节点之间有排斥力（防止重叠）和吸引力（信任越高越靠近）
